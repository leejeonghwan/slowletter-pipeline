# 기초자료 HTML 생성 가이드 (v3.1)

이 문서는 기초자료 HTML 출력의 **데이터 구조와 렌더링 규칙**을 정의합니다. HTML 태그 문제가 아니라 데이터를 구조화하는 방식을 개선하는 것이 핵심입니다.

---

## 원칙

1. **먼저 데이터 구조를 만들고, 그 다음 HTML로 렌더링해주세요.** HTML을 직접 조립하면 제목·인용·링크가 어긋납니다.
2. **하나의 클러스터(또는 기사)에 속하는 정보는 하나의 객체로 묶어주세요.** 제목 따로, 인용 따로, 링크 따로 흩어지면 안 됩니다.
3. **원문 인용은 절대 자르지 마세요.** 여는 따옴표와 주어를 포함한 전체 문장을 유지해주세요.
4. **`<li>` 태그는 반드시 `<ul>` 또는 `<ol>` 안에서만 사용하세요.** `<li>`를 단독으로 쓰면 HTML 규격 위반입니다.
5. **값이 없으면 출력하지 마세요.** `nan`, `None`, 빈 문자열은 HTML에 표시하지 마세요. 링크가 없으면 링크 태그 자체를 생략하세요.
6. **섹션 제목은 반드시 `<h2>` 태그를 사용하세요.** `<p>[주요 키워드]</p>` 형태로 쓰지 마세요.

## 데이터 모델

HTML을 생성하기 전에, 각 섹션의 데이터를 아래 구조로 먼저 정리해주세요.

### 주요 키워드 항목

```python
{
    "title": "노태악 대법관 오늘 퇴임인데, 후임 40일 넘게 감감무소식",
    "papers": ["조선일보", "중앙일보", "한겨레"],
    "conclusion": "\"조 대법원장은 40일이 넘도록 이재명 대통령에게 제청할 후보를 정하지 않고 있다.\"",  # 원문 전체
    "conclusion_source": "조선일보",
    "links": [
        {"paper": "조선일보", "url": "https://..."},
        {"paper": "한겨레", "url": "https://..."}
    ]
}
```

### 쓸고퀄 디테일 항목

```python
{
    "paper": "한겨레",
    "title": "국세청 압류 코인, 한 번 더 털렸다…반납된 뒤 또 탈취",
    "point": "\"국세청의 보도자료 비밀번호 노출로 벌어진 '압류 가상자산 탈취'가 애초 알려진 것과 달리 두 차례 반복해 벌어진 것으로 확인됐다.\"",  # 원문 전체

}
```

### 오늘의 주요 이슈 항목 (요약형)

```python
{
    "title": "이란 '초토화 전략' 전방위 반격…걸프국·유럽까지 전쟁 소용돌이",
    "papers": ["경향신문", "조선일보", "중앙일보", "한겨레"],
    "point": "\"이란은 '미군 기지 등 군사시설만 겨냥했다'고 주장하지만, 공항·호텔·민간 주거 단지까지 피해를 입고 있다.\"",  # 원문 전체
    "point_source": "조선일보",
    "links": [
        {"paper": "조선일보", "url": "https://..."},
        {"paper": "중앙일보", "url": "https://..."},
        {"paper": "한겨레", "url": "https://..."}
    ]
}
```

### 클러스터 상세 항목

```python
{
    "title": "이란 '초토화 전략' 전방위 반격…걸프국·유럽까지 전쟁 소용돌이",
    "papers": ["경향신문", "조선일보", "중앙일보", "한겨레"],
    "summary": "이란이 이스라엘 대신 걸프 국가들을 공격하며...",
    "comments": [  # 중요 코멘트: 실제 사람이 말한 인용문만
        {
            "quote": "\"이란은 '미군 기지 등 군사시설만 겨냥했다'고 주장하지만...\"",
            "speaker": "조선일보",
            "paper": "조선일보",
            "link": "https://..."
        }
    ],
    "one_liner": "미-이란 중재국 오만까지 공격 중동 전역에 미사일 165발·드론 541기 쏴",
    "best_quote": {  # div.point에 표시할 대표 인용
        "quote": "\"UAE 국방부는 전날 이란에서 탄도미사일 165기, 드론 541기가 날아왔으며...\"",
        "speaker": "중앙일보"
    },
    "facts": [  # 숫자/팩트: 통계, 수치, 사실관계만
        {
            "text": "지난해 전 세계 국제선 여객 수 1위(약 9200만명)를 기록한 두바이 국제공항은...",
            "paper": "조선일보",
            "link": "https://..."
        }
    ],
    "quotes": [  # 발언/인용: 실제 사람이 직접 말한 문장만 (기사 리드문·사진 캡션 제외)
        {
            "quote": "\"중동뿐 아니라 세계 평화라는 우리의 목표 달성을 위한 폭격은 계속될 것이다.\"",
            "speaker": "도널드 트럼프",
            "paper": "조선일보",
            "link": "https://..."
        }
    ],
    # ↓ 아래 3개는 facts/quotes와 별도 필드. 절대 facts ul 안에 넣지 마세요.
    "temperature": "조선일보는 '임박한 위협 제거'라는 안보 프레임, 한겨레는 '선택에 의한 전쟁'이라는 비판 프레임으로 다뤘다.",
    "per_paper": {  # 참여 신문 전부 기재
        "조선일보": "이 대통령, 싱가포르와 SMR 공동 개발 MOU 체결 — AI·원전 등 첨단 협력에 초점",
        "한겨레": "한·싱 FTA 20년 만에 개선 협상… — 통상 구조 변화에 초점",
        "경향신문": "이 대통령, '초불확실성 시대의 진정한 동반자' — 대통령 발언 중심"
    },
    "context_keywords": "이란, 걸프 국가, 미사일"
}
```

**핵심 규칙:**
1. `quote` 필드에는 원문 전체를 넣어주세요. 앞부분을 자르면 안 됩니다.
2. `facts`에는 숫자/통계/사실관계만 넣고, `quotes`에는 실제 발언만 넣으세요. **섞지 마세요.**
3. `temperature`, `per_paper`, `context_keywords`는 독립 필드입니다. `facts` 리스트나 `quotes` 리스트 안에 넣지 마세요.
4. `quotes`의 "발언"이란 **사람이 직접 말한 문장**입니다. 기사 리드문("○○일보에 따르면..."), 사진 캡션("○○이 ○일 ○○에서..."), 기사 요약문은 발언이 아닙니다.

## HTML 렌더링 규칙

위 데이터 구조를 HTML로 변환할 때 아래 규칙을 따라주세요.

### 전체 구조

```html
<body>
  <!-- 헤더 -->
  <div class="section">
    <p>슬로우레터 기초 자료</p>
    <p class="meta">기준일 / 생성 시각 / 원칙</p>
  </div>

  <!-- 섹션 1 -->
  <h2>주요 키워드</h2>
  <p class="meta">기준: 4개. 빠르게 핵심 파악용.</p>
  <!-- 키워드 항목들 -->

  <h2>쓸고퀄 디테일</h2>
  <p class="meta">기준: 단독/소수에서 공적 가치/직접 인용/숫자 재료가 강한 것 최대 7개.</p>
  <!-- 디테일 항목들 -->

  <h2>오늘의 주요 이슈</h2>
  <p class="meta">기준: 클러스터 단위로 15개.</p>
  <!-- 이슈 항목들 -->

  <!-- 클러스터 상세 블록들 -->
</body>
```

### 주요 키워드 항목 렌더링

```html
<div class="cluster-item">
  <p><strong>{title}</strong> ({papers 쉼표 구분})</p>
  <div class="point">{conclusion} / {conclusion_source}</div>
  <p class="press-links">
    <a href="{url}" target="_blank">[{paper}]</a>
    <a href="{url}" target="_blank">[{paper}]</a>
  </p>
</div>
```

### 쓸고퀄 디테일 항목 렌더링

각 기사를 개별 `cluster-item`으로 감싸주세요. **링크를 맨 아래에 한꺼번에 모으지 마세요.**

```html
<div class="cluster-item">
  <p><strong>{paper}:</strong> {title}</p>
  <div class="point">{point} / {point_source}</div>
  <p class="press-links"><a href="{url}" target="_blank">[{paper}]</a></p>
</div>
```

### 오늘의 주요 이슈 항목 렌더링

```html
<div class="cluster-item">
  <p><strong>{title}</strong> ({papers 쉼표 구분})</p>
  <div class="point">{point} / {point_source}</div>
  <p class="press-links">
    <a href="{url}" target="_blank">[{paper}]</a>
    <a href="{url}" target="_blank">[{paper}]</a>
  </p>
</div>
```

### 클러스터 상세 렌더링

**중요: 각 섹션(숫자/팩트, 발언/인용, 온도차, 신문별 한줄)은 완전히 분리된 블록입니다. 하나의 `<ul>` 안에 모든 데이터를 섞어 넣지 마세요.**

```html
<div class="section">
  <h3>{title}</h3>
  <p class="press-list">언론: {papers 쉼표 구분}</p>
  <p><strong>[핵심]</strong> {summary}</p>

  <h4>중요 코멘트</h4>
  <ul>
    <li>{quote} / {speaker} — <a href="{link}">[{paper}]</a></li>
  </ul>

  <p><strong>한줄 맥락:</strong> {one_liner}</p>
  <div class="point">{best_quote} / {speaker}</div>

  <h4>숫자/팩트</h4>
  <ul>
    <!-- ⚠️ 여기에는 숫자·통계·사실관계만 넣으세요 -->
    <li>{text} ({paper}) <a href="{link}">[{paper}]</a></li>
  </ul>
  <!-- ⚠️ 숫자/팩트 ul은 여기서 닫힙니다. 아래 내용을 이 안에 넣지 마세요 -->

  <h4>발언/인용</h4>
  <ul>
    <!-- ⚠️ 실제 사람이 말한 직접 인용문만. 기사 리드문·사진 캡션 제외 -->
    <li>{speaker}: {quote} ({paper}) <a href="{link}">[{paper}]</a></li>
  </ul>

  <p><strong>온도차:</strong> {temperature}</p>
  <!-- ⚠️ 신문명을 명시하고 구체적 프레임 차이를 써주세요 -->

  <p><strong>신문별 한줄(차이만):</strong></p>
  <ul>
    <!-- ⚠️ 참여 신문 전부 기재. 1개만 쓰지 마세요 -->
    <li>{신문명}: {그 신문만의 앵글 한줄}</li>
    <li>{신문명}: {그 신문만의 앵글 한줄}</li>
  </ul>

  <p><strong>붙일 맥락(키워드):</strong> {context_keywords}</p>
</div>
```

**절대 하지 말 것:**
- ❌ 숫자/팩트 `<ul>` 안에 `<li>발언/인용(직접 인용, 문장 그대로):</li>`를 넣는 것
- ❌ 숫자/팩트 `<ul>` 안에 `<li>온도차: ...</li>`를 넣는 것
- ❌ 숫자/팩트 `<ul>` 안에 `<li>신문별 한줄(차이만):</li>`를 넣는 것
- ❌ 발언/인용 데이터를 두 번 출력하는 것 (숫자/팩트 안 + 별도 섹션)
- ❌ 기사 리드문("○○에 따르면...")을 발언으로 분류하는 것
- ❌ 사진 캡션("○○이 ○일 ○○에서 기자회견을 하고 있다")을 발언으로 분류하는 것

## 현재 발생하고 있는 문제와 원인

| 문제 | 원인 | 해결 |
|------|------|------|
| 인용문 앞부분 잘림 ("이란은" 누락, "라는"만 남음) | quote 데이터를 추출할 때 문장을 잘라서 저장하고 있음 | quote 필드에 원문 전체를 저장해주세요 |
| 쓸고퀄 디테일에서 링크가 맨 아래에 뭉침 | 기사별로 묶지 않고 제목 리스트와 링크 리스트를 따로 생성함 | 기사 하나 = 객체 하나로 묶은 뒤 렌더링해주세요 |
| 섹션 제목이 시각적으로 구분 안 됨 | `<p>[주요 키워드]</p>` 형태로 사용 | `<h2>주요 키워드</h2>` 태그로 변경해주세요 |
| 링크가 세로로 길게 나열 | `<li>` 안에 링크를 하나씩 넣음 | `<p class="press-links">` 안에 `<a>` 인라인으로 넣어주세요 |
| `<li>` 태그가 `<ul>` 없이 단독 사용 | 목록 태그 규칙을 무시하고 `<li>`를 직접 배치함 | 모든 `<li>`는 반드시 `<ul>` 또는 `<ol>` 안에 넣어주세요. 목록이 아니면 `<p>` 또는 `<div>`를 사용하세요 |
| `nan` 값이 HTML에 표시됨 | 링크가 없는 항목에서 None/nan을 그대로 출력함 | 값이 None이거나 nan이면 해당 요소를 아예 생략해주세요 |
| 주요 키워드·쓸고퀄 항목이 개별 단위로 구분 안 됨 | 항목별 `<div class="cluster-item">` 래핑 없음 | 각 항목을 `<div class="cluster-item">`으로 감싸주세요 |
| 발언자가 항상 "(발언자 미상)"으로 표시됨 | 기사에서 발언자를 추출하지 못함 | 인용문 앞뒤에 발언자명이 있으면 반드시 추출해주세요. 정말 불명일 때만 "(발언자 미상)"을 사용하세요 |
| **발언/인용 데이터가 두 번 출력됨** | 숫자/팩트 `<ul>` 안에 발언/인용 데이터를 넣고, 별도 `<h4>발언/인용</h4>` 섹션에 같은 내용을 또 출력함 | 숫자/팩트와 발언/인용은 **별도 섹션으로 완전히 분리**해주세요. 숫자/팩트 ul 안에 발언/인용, 온도차, 신문별 한줄, 붙일 맥락을 넣지 마세요 |
| **온도차·신문별 한줄·맥락이 `<li>`로 숫자/팩트 안에 섞임** | 데이터 구조를 분리하지 않고 하나의 ul에 모든 항목을 나열함 | 온도차, 신문별 한줄, 붙일 맥락은 각각 독립된 `<p>` 또는 `<h4>` 섹션으로 분리해주세요 |
| **기사 리드문·사진 캡션을 "발언"으로 수집** | 기사 첫 문장이나 사진 설명을 인용문으로 잘못 분류함 | "발언/인용"에는 **실제 사람이 직접 말한 문장**만 넣어주세요. 기사 리드문("○○일보에 따르면...")이나 사진 캡션("○○이 ○일 ○○에서...")은 발언이 아닙니다 |
| **클러스터 간 데이터 오염** | 날씨 클러스터에 축구 데이터, 인터뷰 클러스터에 도로·건강 데이터가 섞임 | 클러스터링 후 각 클러스터의 기사가 **같은 주제**인지 검증해주세요. 관련 없는 기사가 섞이면 제거하세요 |
| **한줄 맥락이 불완전한 문장으로 잘림** | 긴 문장을 그대로 넣어 중간에서 끊김 ("...말했 조 대법원장의...") | 한줄 맥락은 **직접 작성한 요약문** 한 문장이어야 합니다. 기사 원문을 복사-붙여넣기하지 마세요 |

## 내용 품질 기준

HTML 구조뿐 아니라 **데이터 내용의 품질**도 중요합니다. 아래 기준을 지켜주세요.

### 발언자 추출

- 인용문 앞뒤에 발언자명이 있으면 **반드시 추출해주세요.**
- 기자 칼럼/시각/왜냐면 등은 **기자 이름**이 발언자입니다.
- 중요 코멘트에서 발언자를 잘 추출했다면, 같은 기사의 "발언/인용" 섹션에서도 동일하게 추출해주세요.
- **정말 확인 불가능한 경우에만** "(발언자 미상)"을 사용하세요. 전체 항목의 절반 이상이 "(발언자 미상)"이면 추출 로직을 점검해주세요.

### 온도차 분석

피상적인 분석은 쓸모가 없습니다. 아래처럼 **구체적인 프레임 차이**를 써주세요.

**나쁜 예:**
> "프레임의 차이는 양국의 경제적 연대와 안보 협력의 필요성을 강조하는 점에서 나타난다."

**좋은 예:**
> "조선일보는 '임박한 위협 제거'라는 안보 프레임, 한겨레는 '선택에 의한 전쟁'이라는 비판 프레임으로 다뤘다. 경향신문은 이란 측 입장을 상대적으로 많이 인용했다."

온도차는 **신문명을 명시하고, 각 신문이 어떤 앵글로 다루었는지** 구체적으로 적어주세요.

### 신문별 한줄 (차이만)

"신문별 한줄"은 각 신문의 **차별적 관점**을 보여주는 섹션입니다. 하나의 요약문만 적으면 안 됩니다.

**나쁜 예:**
> "양국은 FTA 개선 협상 개시 합의 공동선언문을 발표했다."

**좋은 예:**
> - 조선일보: "이 대통령, 싱가포르와 SMR 공동 개발 MOU 체결" — AI·원전 등 첨단 협력에 초점
> - 한겨레: "한·싱 FTA 20년 만에 개선 협상… 공급망·녹색경제 4개 분야" — 통상 구조 변화에 초점
> - 경향신문: "이 대통령, '초불확실성 시대의 진정한 동반자'" — 대통령 발언 중심

신문마다 한줄씩, **그 신문만의 앵글**을 적어주세요.

### 한줄 맥락

- **사진 캡션을 한줄 맥락으로 쓰지 마세요.** "임우형 LG AI연구원 공동 연구원장(오른쪽)과..."는 사진 설명이지 맥락이 아닙니다.
- 한줄 맥락은 해당 클러스터가 **왜 중요한지, 어떤 흐름 속에 있는지**를 한 문장으로 요약한 것입니다.

**나쁜 예:**
> "임우형 LG AI연구원 공동 연구원장(오른쪽)과 이상엽 LG유플러스 CTO가 1일 스페인 바르셀로나에서 기자간담회를 갖고 있다./뉴시스"

**좋은 예:**
> "MWC 2026 개막, 국내 통신 3사 AI 전환 경쟁 본격화"
