## 핵심 프로젝트

### 1. 신문 지면 분석 (newspaper_clusters_materials)

**목적:** 4대 신문 지면 기사 → 구조화된 HTML 자료
- 수집: `naver_newspaper_full_collect.py`
- 분석: `generate_materials_v2.py` (v3.1)
- 출력: HTML (팩트/발언 완전 분리)

**핵심 원칙 (OPENCLO_HTML_GUIDE v3.1):**
1. 섹션 완전 분리 (숫자/팩트 ↔ 발언/인용)
2. 발언자 미상 제거 (유효 발언자만)
3. 사진 캡션 필터링
4. 온도차 = 신문별 프레임 차이 명시
5. 원문 인용 절대 자르지 않기

**품질 검증 자동화:**
- "(발언자 미상)" 0건
- 섹션 중복 차단
- 캡션 혼입 차단

## 배운 교훈

1. **"Mental notes" 금지**
   - 기억은 파일로만 (MEMORY.md, memory/YYYY-MM-DD.md)
   - 세션 재시작 시 모든 기억 날아감

2. **품질 > 속도**
   - 이정환님: "시간은 중요하지 않습니다. 최선의 결과를 주세요."
   - 빠르게 대충 보다, 느려도 정확하게

3. **발언자 확인 불가 = 그냥 버리기**
   - "(발언자 미상)" 남발하지 않기
   - 익명 출처도 가치 있으면 포함 (단, 명확히 표기)

4. **깃허브 = 외부 뇌**
   - 리셋 대비 모든 컨텍스트 동기화
   - .gitignore로 개인정보 보호 (memory/, USER.md)
